# Test Idea Template

Use this template to submit test ideas for ZIME patent validation.

---

## Instructions

1. Copy this template
2. Create a new file: `YYYY-MM-DD_your_name_idea.md`
3. Fill in all sections
4. Submit via pull request or issue

---

## Template

```markdown
# Test Idea: [Your Title Here]

**Submitted by:** [Your Name/Handle]  
**Date:** [YYYY-MM-DD]  
**Contact:** [Optional - email or GitHub handle]

## Hypothesis

What do you expect this test to prove or disprove?

Example: "I hypothesize that ZIME's PSI detection will fail under extremely high-frequency signal changes (>1MHz)."

## Test Description

Describe the test in detail:
- What inputs?
- What parameters?
- What environment?
- What measurements?

## Success Criteria

What results would validate your hypothesis?

Example: 
- If PSI detection rate drops below 10% at 1MHz signals â†’ hypothesis validated
- If PSI detection maintains >25% at 1MHz â†’ hypothesis refuted

## Failure Criteria

What results would refute your hypothesis?

## Relevance to Patent

Which patent claim does this test support or challenge?

Reference: See `PROVISIONAL_2/CLAIMS.md` for claim list.

## Additional Notes

Any other context, concerns, or suggestions.
```
